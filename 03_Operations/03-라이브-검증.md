# 라이브 검증

## 목적

실제 운영 환경에서 route, delivery, keyword detection이 의도대로 동작하는지 확인한다.

## 원본 소스 포인터

- `clawhip/docs/live-verification.md`
- `clawhip/scripts/live-verify-default-presets.sh`

## 최소 검증 항목

- daemon health 확인
- custom send 실메시지 확인
- GitHub issue / PR flow 확인
- tmux keyword / stale 감지 확인
- native event contract payload 확인

## 학습 포인트

- mock 테스트와 live verification의 차이
- 운영 검증에서 필요한 사전 조건(bot token, auth, tmux, route filters)
