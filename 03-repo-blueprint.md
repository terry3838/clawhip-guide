# clawhip-guide 저장소 청사진

## 구조 원칙

1. **Frontdoor는 짧고 명확하게**
   - `README.md`는 저장소 목적, 추천 학습 순서, 폴더 구조를 제공한다.
2. **학습 흐름은 단계형으로**
   - 기초 → 내부 구조 → 운영 → 실습 → 참고자료 순으로 이어진다.
3. **원본 소스 포인터를 항상 포함**
   - 각 문서는 대응하는 `clawhip` 원본 파일을 함께 적는다.
4. **문서와 실습을 분리**
   - 설명 문서는 `01_~03_`, 실행 과제는 `04_Labs/`에 둔다.
5. **확장에 강한 번호 체계 사용**
   - 새 문서가 들어와도 중간 삽입이 쉬운 구조를 유지한다.

## 최소 섹션 집합

- 소개 / 문제 정의
- 설치 / 빠른 시작
- 아키텍처 개요
- 이벤트 계약과 라우팅
- CLI와 설정
- Git/GitHub/tmux 운영 연동
- memory / plugins / skills
- live verification
- 실습 랩
- 체크리스트 / 런타임 파일 맵

## 문서 템플릿

각 문서는 아래 5블록을 기본으로 사용한다.

1. 이 문서에서 답할 질문
2. 왜 중요한가
3. 원본 소스 포인터
4. 핵심 설명
5. 실습/체크포인트

## 향후 추가하면 좋은 것

- `assets/diagrams/*.svg` 렌더링 파이프라인
- `examples/` 디렉토리와 sample route config
- 릴리스별 변경점 비교 문서
- OMC/OMX/OpenClaw와의 통합 사례집
